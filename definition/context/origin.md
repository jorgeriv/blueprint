# Origin

Status: active

Blueprint began as a way to support the messy beginning of a project, before the
work is clear enough for implementation, task planning, or traditional project
management.

The initial idea included goals, decisions, and backlog items, but the project
definition shifted during discussion. The key insight was that early-stage
projects need durable context before they need actionable tasks. Blueprint
should therefore produce project documentation and context first.

The project is now using itself as a meta test case: Blueprint is being
initialized inside the Blueprint repository so its own purpose, scope, decisions,
and unresolved questions can be refined through the same process it offers.
