# Principles

Status: active

## Initial Principles

- Start before the project is fully clear.
- Distill conversation instead of storing transcripts.
- Label assumptions as assumptions.
- Separate possibilities from decisions.
- Preserve open questions instead of forcing premature answers.
- Let the initializer own structure and skills own substance.
- Treat backlog and implementation work as downstream exports, not the core
  output.
- Keep the framework useful across domains, not just software projects.
